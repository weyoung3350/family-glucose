## 详细需求

### deploy/README.md
按以下步骤组织（适合用户照做即可上线）：

```markdown
## 一、初次部署
1. 服务器创建用户：`useradd -r -s /bin/false glucose`
2. clone 项目到 /opt/glucose-api
3. cd /opt/glucose-api && python3 -m venv .venv && .venv/bin/pip install -e .
4. cp .env.example .env，填入：
   - JWT_SECRET（运行 `openssl rand -hex 32` 生成）
   - WX_APPID / WX_SECRET（微信公众平台获取）
   - DEEPSEEK_API_KEY（或留空，会自动走规则解析器）
5. 建表：`.venv/bin/python scripts/init_db.py`
6. chown -R glucose:glucose /opt/glucose-api
7. cp deploy/glucose-api.service.example /etc/systemd/system/glucose-api.service
8. systemctl daemon-reload && systemctl enable --now glucose-api
9. cp deploy/nginx.conf.example /etc/nginx/conf.d/glucose-api.conf，编辑域名
10. nginx -t && systemctl reload nginx
11. 微信公众平台：开发管理 → 服务器域名 → request 合法域名加入你的 https://yourdomain
12. 加 cron：`0 3 * * * /opt/glucose-api/deploy/backup.sh`

## 二、升级
1. cd /opt/glucose-api && git pull
2. .venv/bin/pip install -e .
3. systemctl restart glucose-api

## 三、查看日志
- systemd 日志：`journalctl -u glucose-api -f`
- 文件日志：`/var/log/glucose-api/*.log`

## 四、回滚
1. systemctl stop glucose-api
2. cp /var/backups/glucose/glucose-YYYYMMDD.db data/glucose.db
3. git checkout <prev-tag>
4. systemctl start glucose-api
```

### deploy/glucose-api.service.example
```
[Unit]
Description=家有糖人 API
After=network.target

[Service]
Type=simple
User=glucose
Group=glucose
WorkingDirectory=/opt/glucose-api
EnvironmentFile=/opt/glucose-api/.env
ExecStart=/opt/glucose-api/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8080 --workers 1 --no-access-log
StandardOutput=append:/var/log/glucose-api/app.log
StandardError=append:/var/log/glucose-api/error.log
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### deploy/nginx.conf.example
```
# 假设主站点已经存在，把以下 location 加到现有 server 块
location /api/ {
    proxy_pass http://127.0.0.1:8080/api/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 30s;
    proxy_connect_timeout 5s;
    client_max_body_size 1m;
}
```

### deploy/backup.sh
```bash
#!/bin/bash
set -e
DB=/opt/glucose-api/data/glucose.db
DEST=/var/backups/glucose
mkdir -p "$DEST"
ts=$(date +%Y%m%d)
sqlite3 "$DB" ".backup $DEST/glucose-$ts.db"
# 保留 30 天
find "$DEST" -name "glucose-*.db" -mtime +30 -delete
echo "$(date -Iseconds) backup ok: glucose-$ts.db"
```

### deploy/logrotate.conf.example
```
/var/log/glucose-api/*.log {
    daily
    rotate 14
    compress
    missingok
    notifempty
    sharedscripts
    postrotate
        systemctl reload glucose-api > /dev/null 2>&1 || true
    endscript
}
```

## 验收标准

- [ ] 在一台干净的 Ubuntu 22.04 测试机上按 README 步骤部署，最后 `curl https://yourdomain/api/v1/health` 返回 200
- [ ] systemd 状态 `systemctl status glucose-api` 显示 active (running)
- [ ] 重启服务器后服务自启
- [ ] 写一条记录后停服 → 重启 → 记录仍在
- [ ] backup.sh 执行后 /var/backups/glucose/ 出现当日 db 文件
- [ ] 日志轮转：手动 `logrotate -f deploy/logrotate.conf.example` 后 .log 被压缩
