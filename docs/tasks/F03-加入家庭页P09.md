# F03 · 加入家庭页 P09

## 目标
实现 P09 加入家庭页：登录后无家庭归属时进入此页，二选一（输入邀请码加入 / 创建新家庭）。

## 前置依赖
F02、B03、B04

## 上下文
- PRD §6.9（P09 加入家庭）
- 概要设计 §7.1（首次使用流程）
- 原型截图：`docs/assets/家有糖人/P09-加入家庭.png`

## 输入
- 当前用户在 globalData 中无 family（`app.globalData.family === null`）
- 可选 query：`?code=XXXXXX`（来自微信分享卡片）

## 输出

```
miniprogram/
└── pages/
    └── join/
        ├── join.js
        ├── join.wxml
        ├── join.wxss
        └── join.json
```

修改 `app.js` 的 `onLaunch` 增加 family 缺失时的跳转逻辑。

## 详细需求

### 路由跳转
`app.js` 在登录完成后（含本地缓存恢复或 wx.login 回来后）：
- 若 `globalData.family == null` → `wx.reLaunch({ url: "/pages/join/join" + (code ? `?code=${code}` : "") })`
- 否则正常进入 TabBar 首页

### join.wxml 结构
按 P09 原型：
- hero 区：糖字 logo + "家有糖人" + "一家人一起记录爸爸的血糖"
- 邀请码大输入框（dashed border, 大字距 8rpx）
  - 自动转大写，限制 6 位字母数字
  - 如有 query.code 则预填
- 主按钮"加入家庭"
- 分隔线"或"
- 次按钮"+ 创建新家庭"
- 创建按钮点击 → 弹出 weui-modal 输入家庭名 → 提交

### 行为
- 加入：调 `api.joinFamily(code)`：
  - 成功 → 写 globalData.family、wx.setStorageSync("family", ...)、`wx.switchTab("/pages/index/index")`
  - 失败 ERR_INVITE_CODE_INVALID → wx.showToast({title:"邀请码不存在",icon:"none"})
  - 失败 ERR_USER_ALREADY_IN_FAMILY → wx.showToast 同
- 创建：modal 校验 1-20 字符，不允许全空白
  - 调 `api.createFamily(name)`：
    - 成功 → 同上跳首页
    - ERR_FAMILY_NAME_TAKEN → modal 内显示红字"该家庭名已被使用"，不关闭

### join.wxss
按 PRD 视觉规范，字号大、按钮高（96rpx 起）、橙色主按钮。糖字 logo 用渐变背景 `linear-gradient(135deg, #FFB066, #FF9F40)` 圆角 36rpx。

## 验收标准

- [ ] 用户登录后 family=null 自动跳到 /pages/join/join
- [ ] 输入正确的邀请码 → 成功跳首页，globalData.family 已填
- [ ] 输入错误的邀请码 → toast 提示
- [ ] 点"创建新家庭" → modal 弹出，输入"咱家"提交后跳首页
- [ ] 创建同名家庭 → modal 不关，显示红字错误
- [ ] 通过分享卡片打开（路径含 ?code=XXX）→ 邀请码自动预填
- [ ] 输入框自动大写：手动输入 "abc123" 显示为 "ABC123"
- [ ] 输入框限制 6 位（多余被忽略）
- [ ] 视觉与 P09 原型截图差异不大于 10%（字号、间距、颜色）

## 不在范围内
- 微信分享出 invite_code（在 F09 的家庭管理页实现）
- 退出家庭后再次进入此页（已涵盖在路由逻辑里）
