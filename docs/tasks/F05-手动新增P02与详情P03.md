## 详细需求

### P02 add 页

#### 顶部分段切换
两个 tab："手动录入" / "AI 一句话"。点击"AI 一句话"立即 `wx.redirectTo` 到 `/pages/ai-add/ai-add`（不返回手动）。

#### 字段
- 血糖值
  - 自定义大字号输入区（80rpx 字号），点击呼出 `<input type="digit">` 隐藏键盘
  - 单位 mmol/L 不可改
  - 输入限制 `0 < value < 50`，保留 1 位小数
- 时段 chips（period-picker 组件，横排可换行）
- 测量时间
  - `<picker mode="multiSelector">` 自定义日期+时间，时间步进 5 分钟（小时 0-23、分钟 [00,05,10,15,20,25,30,35,40,45,50,55]）
  - 默认值：当前时间 round_to_5min
- 备注 `<input maxlength="200">`

#### 底部固定保存按钮
- 高度 96rpx 主按钮，文字"保存"
- 必填校验失败 toast；通过则调 `api.createRecord`
- 编辑模式（query 含 `?id=`）：读 detail 数据填表，保存调 `api.updateRecord`

#### 完成后
`wx.navigateBack()`，触发首页 onShow 重新拉数据。

### P03 detail 页

#### 顶部
导航栏标题"记录详情"，右上更多按钮（仅记录人本人显示，弹出 actionSheet：编辑、删除）。

#### Hero 区
大数字血糖值（status 色） + 单位 mmol/L + 状态胶囊（理想/一般等） + 副文字"理想范围（餐后）4.4 ~ 7.8"

#### Meta 区
- 时段 → 中文
- 测量时间 → YYYY-MM-DD HH:mm
- 记录人 → 头像 + 昵称
- 记录时间 → created_at
- 备注 → note 或"无"
- 来源标记：source=='ai' 时显示一个小标签"AI 录入"

#### 操作区
仅记录人本人可见：
- "编辑" 按钮 → `wx.navigateTo("/pages/add/add?id=...")`
- "删除" 按钮 → 二次确认 modal → `api.deleteRecord(id)` → `wx.navigateBack()`

非记录人本人不显示这两个按钮，仅显示返回手势。

### components/glucose-chip
简单组件，props: `level` （low/ideal/...）和 `label`，渲染对应背景色 + 白字胶囊。

### components/period-picker
props: `value`（当前选中 key），`bindchange` 抛出新 key。横排，每个 chip 圆角 36rpx，选中态橙色填充白字。

## 验收标准

### P02
- [ ] 输入血糖 7.2 / 选时段早餐后 / 时间默认 round 到 5 分钟 / 备注 "测试"，保存成功后回首页能看到这条记录
- [ ] 时间 picker 的分钟列只有 12 个选项（00-55，步进 5）
- [ ] 输入 0 或 51 提示错误，不写库
- [ ] 编辑模式：从详情页跳过来，表单已预填，保存后详情页数据更新
- [ ] 顶部"AI 一句话"点击跳转 P02b

### P03
- [ ] 进入详情看到大数字血糖、状态色胶囊、4 行 meta
- [ ] AI 录入的记录显示"AI 录入"小标签
- [ ] 自己记录的有"编辑/删除"，他人记录的没有
- [ ] 删除后回到列表，该条不再出现
- [ ] 编辑后回到详情，新数据立即更新
