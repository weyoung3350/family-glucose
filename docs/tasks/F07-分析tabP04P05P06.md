## 详细需求

### 分段控制
顶部三选一："按天矩阵 / 趋势图 / 报表"。

### 时间范围选择
通用：weui-style picker，选项"最近 7 天 / 最近 30 天 / 最近 90 天 / 自定义"。
默认值：矩阵 7 天，趋势图 30 天，报表 30 天。

### 子视图 1：按天矩阵
- 顶部时间范围 picker
- 表格：横轴 8 时段（用 shortLabel：空腹/早后/午前/...），纵轴日期倒序
- 单元格：有数据时显示彩色块（status.color 背景 + 白字 value），无数据显示灰底"-"
- 单元格 tap → 抽取该天该时段的最新一条 → `wx.navigateTo("/pages/detail/detail?id=...")`

### 子视图 2：趋势图
- 时段筛选 chips："全部时段 / 空腹 / 餐后"
  - "餐后"为 after_breakfast/after_lunch/after_dinner 三选一的 OR 查询。**首版简化**：只允许 "全部 / 空腹 / 餐后(三个合并)"，本地组合三次请求或后端支持 period=after_*，由 B06 决定。**采用客户端过滤**：始终拉全部 period，前端按选中筛选。
- 时间范围 picker
- 折线图：
  - 用 `<canvas>` 自绘 或 引入 `wx-charts` 库（首版采用自绘 SVG/canvas，避免引依赖。坐标轴、点、线、理想区间阴影按原型 P05 实现）
  - X 轴：日期等距点
  - Y 轴：0 到 max(values, 14)，0 / 4 / 8 / 14 四个刻度线，4-8 区间绿色阴影代表理想范围（用 family.standards.fasting_low/high）
  - 点的颜色：按各点 status 着色
- 下方统计卡片：测量次数、平均值、最高/最低、理想占比
- 分布柱条：5 段水平条，宽度 = 各档计数占比 × 总宽

### 子视图 3：报表
- 顶部彩色 hero 卡：橙渐变背景，平均值大号 + "偏高/理想..." 状态文字
- 三个数字：总测量数、理想占比、异常天数
- 五档分布柱条
- 按时段表：8 行（按 PRD 顺序），每行显示 `时段：avg · X次 · 状态`，颜色按状态

### 状态保持
切换分段后维持各自的时间范围与筛选条件（用 data 三个独立字段而非共享 currentRange）。

## 验收标准

### 矩阵
- [ ] 选最近 7 天，矩阵显示 7 行，每行 8 列
- [ ] 有记录的格子显示对应颜色与数值；无记录显示灰底 "-"
- [ ] 点格子跳详情

### 趋势图
- [ ] 折线图能正确画出（移动端浏览测试）
- [ ] 选"空腹"后 points 仅含 fasting 记录
- [ ] 统计卡片数字与后端 chart.stats 一致
- [ ] 分布柱条 5 段宽度 = 各档计数占比

### 报表
- [ ] hero 平均值精度 1 位小数
- [ ] 异常天数与后端报告一致
- [ ] 8 时段全部出现（无记录的也显示 "0 次 · -"）

### 视觉
- [ ] 与 P04/P05/P06 原型差异 ≤ 15%（图表元素允许差异更多）

## 不在范围内
- 自定义日期范围 picker（首版只用预设 7/30/90 天，自定义后续）
- 趋势图数据点超过 1000 时的虚拟列表
- 图表交互（点击点显示气泡）
