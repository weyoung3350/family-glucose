## 详细需求

### P08/P08b family 页

进入时调 `api.getFamily()` 拿数据，根据 `family.role_of_me` 渲染不同视图。

#### 管理员视图（role_of_me === "creator"）
- 家庭名卡片：左 "家庭名" 灰字 + 大字名称，右 "编辑" 浅橙文字 → 弹 modal 改名
- 邀请码卡片：
  - 标签"邀请码 · 家人输入即可加入（永久有效，可随时分享）"
  - 大号字距字母数字 6 位（橙色等宽字）
  - 两个按钮 grid 布局：
    - "复制" → `wx.setClipboardData({data: code})` + toast
    - "分享到微信" → 用 `<button open-type="share">`，触发 `onShareAppMessage` 返回 `share.shareInviteCode(family)`
- 成员卡片：
  - 标题"家庭成员（N）"
  - 列表：每行头像 + 名字 + 角色标签 + 加入时间；非自己的行右侧"移除"按钮 → 二次确认 → `api.removeMember`
- 底部"解散家庭"按钮（红字红边）→ 二次确认 modal "解散后所有家人无法再访问数据，确认吗？" → `api.dissolveFamily` → 跳 P09

#### 成员视图（role_of_me === "member"）
- 家庭名卡片：仅展示名称，无编辑
- 角色卡片：标签"我的角色" + "成员 · 可记录、可查看，不可管理"
- 成员卡片：同上但**无移除按钮**
- 底部"退出家庭"按钮（红字红边）→ 二次确认 → `api.leaveFamily` → 跳 P09
- 底部小字提示"如需邀请新家人，请联系管理员"

### P10 standards 页

进入调 `api.getStandards()`。

UI：
- 顶部如果是管理员 → 浅橙提示卡片"管理员 你可以编辑下方阈值，全家成员同步生效"
- 三个分组（"空腹理想区间"、"餐后理想区间"、"警戒值"）
- 每组用白底圆角 list，每行：左标签 + 右数值（橙色，可 tap，弹 number picker 调整步进 0.1）
- 警戒值组：偏低警戒值用蓝色，过高警戒值用红色
- 底部说明灰字："默认值依据中国 2 型糖尿病防治指南，请遵医嘱调整。单位 mmol/L。"
- 底部"保存"主按钮（仅管理员可见）→ 校验后调 `api.updateStandards`

成员视图：
- 不显示顶部提示卡片
- 数值不可点击修改（不响应 tap）
- 不显示保存按钮

校验：
- `critical_low < fasting_low < fasting_high < critical_high`
- `critical_low < postprandial_low < postprandial_high < critical_high`
- 任一失败 toast "上下限不正确，请检查"

## 验收标准

### P08
- [ ] 管理员看到完整邀请码与所有按钮；成员看不到邀请码、看不到移除按钮
- [ ] 复制按钮触发后剪贴板有邀请码
- [ ] 分享按钮使用 button open-type="share"，分享卡片标题正确（带家庭名）
- [ ] 管理员移除成员后，成员列表减少；该成员重新打开小程序进入 P09
- [ ] 管理员解散家庭后跳 P09
- [ ] 成员退出家庭后跳 P09
- [ ] 视觉差异 ≤ 10%

### P10
- [ ] 默认值正确显示 4.4 / 6.1 / 4.4 / 7.8 / 3.9 / 13.9
- [ ] 管理员改 fasting_high 到 6.5 后保存成功
- [ ] 改成 fasting_low=5, fasting_high=4 → toast 错误，不保存
- [ ] 成员进入页面所有数字不可点击，无保存按钮
- [ ] 视觉差异 ≤ 10%

## 不在范围内
- 多管理员（首版唯一）
- 转让管理员
- 标准变更对历史颜色的回填
