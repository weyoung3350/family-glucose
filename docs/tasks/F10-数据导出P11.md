实现 P11 数据导出页：选择时间范围 → 显示将导出的预览（条数）→ 触发后端 CSV 下载并通过微信分享。

## 详细需求

### 时间范围选择
- 三个预设按钮："最近 7 天 / 最近 30 天 / 最近 90 天"，单选；默认 30 天
- 下方 "自定义日期" 行 → tap 打开自定义双 picker（startDate, endDate），覆盖预设

### 预览卡片
进入页面或切换范围后，调 `api.listRecords({ from, to, page:1, size:1 })` 拿 `total` 字段拼装预览：

```
将导出
· 时间范围：YYYY-MM-DD ~ YYYY-MM-DD
· 共 N 条记录
· 字段：日期 / 时间 / 时段 / 血糖值 / 状态 / 记录人 / 备注
· 格式：CSV（可用 Excel 打开）
```

### 生成与分享
点击"生成并分享 CSV"主按钮：
1. 调 `wx.downloadFile({ url: api.csvUrl({from,to}), header:{ Authorization: "Bearer "+token } })` 拿临时文件
2. 成功后 `wx.openDocument({ filePath, fileType:"csv", success: ... })` 让用户预览，或直接 `wx.shareFileMessage({ filePath })` 弹微信分享面板

> 注意：`api.csvUrl` 在 F02 拼了 token 到 query，但 wx.downloadFile 不能改 header，所以用 `api.csvUrl` 直接放 URL 中（API_BASE 已包含），并让后端在 query 中可选接受 `?token=xxx` 作为 fallback。**如后端不接受 query 中的 token**，则改用 `wx.request({...responseType:"arraybuffer"})` 自行落盘后再分享。**首版**：让 B08 在 GET /export/csv 同时接受 `Authorization: Bearer` header 与 `?token=` query，保证下载链路通畅。

下载后：
- iOS：`wx.shareFileMessage({ filePath })` 弹起转发面板
- Android：同 API
- 失败 toast 显示

### 文案
底部小字："生成后可直接转发给医生或保存到聊天"。

## 验收标准

- [ ] 默认进入显示最近 30 天预览，total 与该范围实际条数一致
- [ ] 切换 7 天/90 天后 total 正确变化
- [ ] 点击"生成并分享 CSV" → 微信分享面板弹起，可发送给好友
- [ ] 接收方下载到电脑用 Excel 打开，中文表头无乱码
- [ ] 接收方文件第一行是中文表头
- [ ] 没有任何记录的家庭点击 → 仍然下载（空 CSV，仅表头），toast 提示"没有记录"
- [ ] 视觉差异 ≤ 10%

## 不在范围内
- PDF 导出
- 自定义字段选择
- 自定义日期 picker 的真实实现可放第二阶段（首版 7/30/90 三选一即可）
