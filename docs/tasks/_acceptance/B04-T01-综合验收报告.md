## 非问题：测试脚本中 1 项失败的原因

Claude 自己写的 e2e 测试脚本里有一项 `report.summary.count >= 20` 失败。**这是 Claude 测试脚本本身的 bug，不是 codex 代码问题**：

- Claude 用 raw `sqlite3 INSERT` 插入测试数据时用 `datetime.isoformat()`（产生 `2026-04-10T08:00:00`，T 分隔符）
- SQLAlchemy 通过 ORM 写入时用空格分隔（`2026-04-10 08:00:00.000000`）
- SQLite 用文本字典序比较 datetime；`'T'` (ASCII 84) > `' '` (ASCII 32)
- 所以 `WHERE measured_at <= '2026-04-10 23:59:59.999999'` 会**排除** `'2026-04-10T08:00:00'`

**生产路径所有写入都走 API（SQLAlchemy）**，统一空格分隔，不存在此问题。我用 API 路径补测后 `Apr 10 only report.count = 1` 完全正确。
