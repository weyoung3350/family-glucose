# v0.2 Backlog

> 来源：T01 UX 评估、Sprint 0/1/2 验收结论与 v0.1 封板讨论。
> v0.1 已封板，不再把以下事项塞回当前版本；后续启动 v0.2 时从本文重新排优先级。

## v0.1 已完成基线

- 后端：家庭、用户、血糖记录、自定义标准、分析、CSV 导出、AI 解析、JWT 鉴权与家庭数据隔离。
- 前端：加入家庭、首页、手动/AI 同页录入、详情、分析、我的、家庭管理、自定义标准、导出、关于页。
- 体验增强：五档颜色与文字对比度、AI 角标、后端今日计数、离线暂存、矩阵触达、底部 safe-area、异常高值二次确认、时段二段选择。
- 验证：后端 17 项 grading 单测通过；Sprint 2 自动化验收 11/11 通过；Sprint 0/1 回归仍通过。

## 建议启动原则

- 优先做影响数据模型、安全边界、错误治理和真实使用效率的事项。
- 每个 v0.2 Sprint 控制在 3-5 个主题，避免再次混入碎片化 UI 小修。
- 涉及后端字段或迁移的任务必须先写 API 契约与 SQLite 迁移方案。
