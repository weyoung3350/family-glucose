# 本机调试指南

> 适用：macOS / Linux + Python 3.11+ + 微信开发者工具
> 目标：5 分钟内在本机把后端 + 小程序跑起来，端到端打通登录、记录、查询、AI 解析

## 1. 启动后端（一条命令）

打开终端：

```bash
cd /Users/dna/Documents/Develop/claude_prj/血糖记录/backend
bash scripts/dev_run.sh
```

脚本会自动：
- 检查 Python 版本
- 创建 `.venv` 虚拟环境
- 装依赖
- 生成 `.env`（含随机 `JWT_SECRET`、开发模式 mock 登录）
- 建 SQLite 表
- 启 uvicorn 监听 `127.0.0.1:8080`

启动成功后看到：
```
════════════════════════════════════════════════════════
 后端启动：http://localhost:8080
 健康检查：curl http://localhost:8080/api/v1/health
 文档：    http://localhost:8080/docs
 按 Ctrl+C 停止
════════════════════════════════════════════════════════
INFO:     Uvicorn running on http://127.0.0.1:8080
```

**新开一个终端**验证：
```bash
curl http://localhost:8080/api/v1/health
# {"status":"ok","app":"家有糖人","version":"0.1.0","env":"development"}
```

后端就绪。

## 2. 配置微信开发者工具

打开微信开发者工具，导入项目：
- 项目目录：`/Users/dna/Documents/Develop/claude_prj/血糖记录/miniprogram`
- AppID：填你已有的测试小程序 AppID（个人申请的也行）
- 不勾选"使用云服务"

进入项目后：

### 2.1 关闭域名校验

工具左上角 → "详情" → "本地设置" → 勾选：

- ☑ **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**

这样小程序才能调你的本地 `http://localhost:8080`。**真机调试时这个开关无效，必须用 HTTPS 域名**——本机调试只用模拟器。

### 2.2 检查 API_BASE 已切到本地

`miniprogram/utils/const.js` 顶部应该看到：
```javascript
const USE_LOCAL = true
const API_BASE = USE_LOCAL
  ? 'http://localhost:8080/api/v1'
  : 'https://YOUR_DOMAIN/api/v1'
```

`USE_LOCAL = true` —— 已经是本地调试模式。**上线前改回 false 并填生产域名**。

### 2.3 模拟器开始预览

工具左侧"模拟器"刷新即可。第一次打开会进入"加入家庭" P09 页面。

## 3. 模拟登录与多用户

`.env` 里默认 `WX_MOCK_OPENID=local_dev_mama`。**这意味着所有 wx.login 都会被后端识别成同一个 openid，登录后是同一个用户**。

要模拟多人协作（妈妈、女儿、儿子），有两种办法：

### 方法 A：改 .env 重启（最简单）

每次切换"身份"前修改 `.env`：
```bash
WX_MOCK_OPENID=local_dev_mama   # 妈妈
WX_MOCK_OPENID=local_dev_nuer   # 女儿
WX_MOCK_OPENID=local_dev_erzi   # 儿子
```
改完按 Ctrl+C 停服 → `bash scripts/dev_run.sh` 重启。微信开发者工具里 `清除全部缓存`（清掉本地 token）后重新进入。

### 方法 B：用三个微信开发者工具账号（不推荐）

工具支持多账号但配置麻烦。家庭场景下方法 A 足够。

## 4. 端到端打通流程（3 分钟）

按这个顺序点一遍，覆盖核心闭环：

1. **创建家庭**：进入 P09 → 输入"咱家"创建 → 跳到首页，看到右上"今日已测 0"
2. **加管理员视图**：底部"我的"→ "家庭管理" → 看到 6 位邀请码 + "复制" + "分享到微信"
3. **复制邀请码**记下来（比如 A3K9F2），关闭后端，改 `WX_MOCK_OPENID=local_dev_nuer`，重启后端，工具里"清除全部缓存"重新进入
4. **女儿加入**：进入 P09 → 选"加入家庭" → 输入邀请码 A3K9F2 → 跳首页
5. **女儿记一条**：底部"+ 记一次血糖" → 输 5.8 / 早餐前 / 现在 → 保存
6. **试一下 AI 录入**：首页底部圆形麦克风 → 输入"今天早上 9:20 空腹血糖 11.1，吃了二甲双胍" → 800ms 后看到识别卡片 → 确认保存
7. **看分析**：底部"分析" → 切到"按天矩阵"看到 2 条 / "趋势图"看折线 / "报表"看平均值
8. **导出**：底部"我的" → "数据导出" → 30 天 → 看到将导出 N 条 → 点"生成并分享 CSV"（开发者工具里会下载到本地）

如果这 8 步全跑通，主链路就稳了。

## 5. 常见问题

### 后端启动报"port 8080 already in use"
```bash
lsof -i :8080            # 看占用
kill -9 <PID>            # 杀掉
# 或换端口：
uvicorn app.main:app --reload --port 8090
# 同时改 const.js 里 API_BASE 的端口
```

### 小程序提示"request:fail url not in domain list"
工具的"本地设置"里**勾选"不校验合法域名"**没勾上。

### 小程序提示"网络异常"
1. 后端确实在跑：`curl http://localhost:8080/api/v1/health`
2. 控制台看具体报错（工具里 → "调试器"→ "Network" 标签）
3. CORS 报错：`.env` 里 `CORS_ORIGINS` 加上工具的 origin（一般 dev_run.sh 已经包含 localhost / 127.0.0.1）

### 改了后端代码不生效
`uvicorn --reload` 是开启的，改 .py 文件应该自动重载。如果没生效就 Ctrl+C 重启。

### 改了小程序代码不生效
工具左上角"编译"按钮强制重新编译，或勾选"自动编译"。

### 想清空所有数据重来
```bash
cd backend
rm -f data/glucose.db
python scripts/init_db.py
# 再重启 uvicorn
```

## 6. 调试常用命令

### 直接调后端 API（不通过小程序）
```bash
# mock 登录拿 token
curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"code":"any","nickname":"妈妈"}' | python3 -m json.tool

# 拷贝返回的 token 拼到下一个请求
TOKEN="<上面那个 token>"
curl -s http://localhost:8080/api/v1/families/me \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool
```

### 看 SQLite 数据
```bash
sqlite3 backend/data/glucose.db
> .tables
> SELECT * FROM users;
> SELECT * FROM glucose_records ORDER BY measured_at DESC LIMIT 5;
> .exit
```

### 查 OpenAPI 文档（调试 API 很方便）
浏览器打开 `http://localhost:8080/docs`，能看到所有接口、参数、响应样例，可以直接试。

## 7. 上线前要改的 3 处

1. `.env` 中：
   - `APP_ENV=production`
   - 填真实 `WX_APPID` / `WX_SECRET`
   - 清空 `WX_MOCK_OPENID`
   - 填 `DEEPSEEK_API_KEY`、`LLM_PROVIDER=deepseek`
2. `miniprogram/utils/const.js`：`USE_LOCAL = false` 并把 `'https://YOUR_DOMAIN/api/v1'` 改成你的真实域名
3. `miniprogram/project.config.json`：填正式小程序 AppID
