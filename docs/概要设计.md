## 1. 文档目的

将 PRD 的产品需求翻译为可被开发任务直接消费的技术规约。本文档定义系统总体架构、模块划分、数据模型、API 契约、关键算法、第三方依赖与部署拓扑。后续每个开发任务（位于 `tasks/` 目录）都以本文档为唯一技术参考。

## 6. API 设计

所有接口前缀 `/api/v1`。除 `POST /auth/login` 外均需 `Authorization: Bearer {jwt}`。响应统一 JSON，错误形如 `{"code": "ERR_XXX", "message": "..."}`。

### 6.1 鉴权

| 方法 | 路径 | 请求 | 响应 200 |
| --- | --- | --- | --- |
| POST | /auth/login | `{ code, nickname?, avatar_url? }` | `{ token, user, family }` family 可为 null |

错误：`ERR_WX_CODE_INVALID`、`ERR_WX_API`

### 6.2 家庭

| 方法 | 路径 | 请求 | 响应 200 | 限定 |
| --- | --- | --- | --- | --- |
| POST | /families | `{ name }` | `{ family, members }` | 用户当前未加入家庭 |
| POST | /families/join | `{ invite_code }` | `{ family, members }` | 用户当前未加入家庭 |
| GET | /families/me | – | `{ family, members }` | 已加入 |
| PATCH | /families/me | `{ name }` | `{ family }` | 仅 creator |
| DELETE | /families/me/members/{user_id} | – | 204 | 仅 creator，不能踢自己 |
| POST | /families/me/leave | – | 204 | 仅 member |
| DELETE | /families/me | – | 204 | 仅 creator |

错误：`ERR_FAMILY_NAME_TAKEN`、`ERR_INVITE_CODE_INVALID`、`ERR_FAMILY_NOT_FOUND`、`ERR_PERMISSION_DENIED`、`ERR_USER_ALREADY_IN_FAMILY`

### 6.3 自定义标准

| 方法 | 路径 | 请求 | 响应 200 | 限定 |
| --- | --- | --- | --- | --- |
| GET | /families/me/standards | – | `{ standards }` | 已加入 |
| PATCH | /families/me/standards | `{ fasting_low?, fasting_high?, postprandial_low?, postprandial_high?, critical_low?, critical_high? }` | `{ standards }` | 仅 creator |

校验：`critical_low < fasting_low < fasting_high < critical_high`，且 `postprandial_low < postprandial_high < critical_high`，所有值 > 0 且 < 50。

### 6.4 血糖记录

| 方法 | 路径 | 请求 | 响应 200 |
| --- | --- | --- | --- |
| GET | /records | query: `from`, `to`, `period`, `page=1`, `size=20` | `{ items: [...], total, page, size }` |
| POST | /records | `{ value, period, measured_at, note?, source? }` | `{ record }` |
| GET | /records/{id} | – | `{ record }` |
| PATCH | /records/{id} | 同 POST 子集 | `{ record }` |
| DELETE | /records/{id} | – | 204 |

`record` 序列化包含字段：`id, value, unit("mmol/L"), period, period_label, measured_at, note, source, recorder: { id, nickname, avatar_url }, status: { level: low|ideal|ok|high|vhigh, label: 偏低|理想|一般|偏高|过高, color }, created_at, updated_at`。

PATCH/DELETE 仅允许 `recorder_id == current_user.id`。

`measured_at` 必须为 5 分钟整刻度（00/05/10…/55）；不满足由后端取整到最近 5 分钟。

### 6.5 分析

| 方法 | 路径 | 请求 | 响应 200 |
| --- | --- | --- | --- |
| GET | /analytics/matrix | query: `from`, `to` | `{ days: [{ date, cells: { fasting, after_breakfast, ... } }] }` 每个 cell 为 `{ value, status } 或 null` |
| GET | /analytics/chart | query: `from`, `to`, `period?` | `{ points: [{ measured_at, value, status }], stats: { count, avg, max, min, distribution: { low, ideal, ok, high, vhigh } } }` |
| GET | /analytics/report | query: `from`, `to` | `{ summary: { count, avg, status, abnormal_days, ideal_pct }, distribution, by_period: [{ period, period_label, avg, count, status }] }` |

### 6.6 AI 解析

| 方法 | 路径 | 请求 | 响应 200 |
| --- | --- | --- | --- |
| POST | /ai/parse-record | `{ text }` | `{ parsed: { value?, period?, period_inferred, measured_at?, measured_at_inferred, note? }, missing: [...], raw_text }` |

响应 `missing` 列出关键缺失字段（value / period 二选一缺失即列出）。`period_inferred / measured_at_inferred` 标识该字段是推断而非显式提取。

### 6.7 导出

| 方法 | 路径 | 请求 | 响应 200 |
| --- | --- | --- | --- |
| GET | /export/csv | query: `from`, `to` | text/csv 文件流，附 `Content-Disposition` |

CSV 列：`date, time, period, period_label, value, unit, status, recorder, note`，UTF-8 BOM，便于 Excel 直接打开。

## 7. 认证与权限

### 7.1 微信登录流程

```
小程序                 后端                          微信开放平台
  │ wx.login()           │                             │
  │ code ────────────────▶                             │
  │                      │ POST jscode2session         │
  │                      │ ──────────────────────────▶ │
  │                      │ ◀────────────── openid +─── │
  │                      │     session_key             │
  │                      │ upsert user                 │
  │                      │ sign JWT                    │
  │ ◀── token, user, family ─                          │
```

JWT 字段：`{ sub: user_id, openid, family_id, role, exp }`，有效期 30 天，使用 HS256。`session_key` **不存储**，只在登录时一次性使用。

### 7.2 鉴权中间件

`get_current_user(token)` 解出 user_id → 查 user → 注入 request state；过期或非法返回 401 `ERR_TOKEN_INVALID`。`require_family(user)` 校验 user.family_id 非空，否则 403 `ERR_NOT_IN_FAMILY`。`require_creator(user)` 在 `require_family` 基础上校验 role=='creator'，否则 403 `ERR_PERMISSION_DENIED`。

### 7.3 家庭范围隔离

所有涉及 family_id 的查询必须强制 `WHERE family_id = current_user.family_id`，由 router 层调用 service 时显式传入，service 不接受外部 family_id 参数。

## 8. AI 解析实现

### 8.1 设计目标

输入一句中文（含数字、时段词、时间表述、备注），输出结构化字段。要求识别准确度优于规则匹配；同时对网络/服务异常有降级路径。

### 8.2 主路径：LLM API（DeepSeek）

调用 `https://api.deepseek.com/v1/chat/completions`，模型 `deepseek-chat`，启用 JSON mode（`response_format={"type": "json_object"}`）。

System prompt 示意：

```
你是中文血糖记录解析器。从用户一句话中抽取以下字段，输出严格 JSON：
{
  "value": <number or null>,         // mmol/L 数值
  "period": <string or null>,        // 8 时段枚举之一
  "measured_at": <"HH:MM" or null>,  // 24 小时制时间，5 分钟取整
  "note": <string or null>           // 用药/饮食/感受备注
}
规则：
- 时段枚举：fasting / before_breakfast / after_breakfast / before_lunch / after_lunch / before_dinner / after_dinner / bedtime
- 用户说"饭后两小时"等同 after_<最近一餐>
- "刚才/现在" → 当前时间
- 不要猜测原文未提及的字段，仅返回 null
```

User prompt：用户原文。

后端拿到 JSON 后做后处理：
- `measured_at` 拼到当天日期，做 5 分钟取整（向最近）；超出当天的告警
- 若 LLM 没给出 period 但给了 measured_at：用第 5.4 节时段窗口推断，标记 `period_inferred=true`
- 若 LLM 没给出 measured_at：取 now，5 分钟取整，`measured_at_inferred=true`
- value 校验范围 (0, 50)

### 8.3 兜底：规则解析器

启用条件：DeepSeek 调用失败、超时（>5s）、或环境变量 `LLM_PROVIDER=none`。

实现要点：
- 数字：`re.search(r'(\d+(?:[.．]\d+)?)\s*(?:mmol/?L)?', text)`，`.` 与中文 `．` 互换
- 时段词典（每个时段对应多个中文别名）
- 时间：`re.search(r'(\d{1,2})[:：点](\d{0,2})', text)`，再处理"早上/上午/下午/晚上"前缀
- 中文数字基础映射（一二三...到 23 为止）
- 备注：去掉以上识别到的片段后剩余的非空文本

兜底解析器的覆盖率目标：60% 以上的常见说法可正确识别 value + period 二者之一。

### 8.4 配置

`.env` 中：
```
LLM_PROVIDER=deepseek          # deepseek | none
DEEPSEEK_API_KEY=sk-xxx
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
LLM_TIMEOUT_SEC=5
```

切换到其他 OpenAI 兼容 API 只需改 BASE_URL 和 API_KEY。

## 9. 关键算法

### 9.1 颜色分级（service/grading.py）

```python
def grade(value: float, period: str, std: Standards) -> str:
    if value < std.critical_low:
        return "low"
    if value >= std.critical_high:
        return "vhigh"
    if period in ("after_breakfast","after_lunch","after_dinner"):
        lo, hi = std.postprandial_low, std.postprandial_high
    else:
        lo, hi = std.fasting_low, std.fasting_high
    if value < lo:
        return "low"
    if value <= hi:
        return "ideal"
    if value < std.critical_high * 0.85:
        return "ok"
    return "high"
```

颜色码：`low #4DA3FF / ideal #52C41A / ok #FAAD14 / high #FA8C16 / vhigh #F5222D`。

### 9.2 5 分钟取整

```python
def round_to_5min(dt: datetime) -> datetime:
    minute = (dt.minute // 5) * 5 + (5 if dt.minute % 5 >= 3 else 0)
    if minute == 60:
        return (dt.replace(minute=0, second=0, microsecond=0) + timedelta(hours=1))
    return dt.replace(minute=minute, second=0, microsecond=0)
```

### 9.3 邀请码生成

字符集 `ABCDEFGHJKLMNPQRSTUVWXYZ23456789`（去 0/O/1/I/L），长度 6，使用 `secrets.choice`。冲突重试 5 次仍冲突则报错。

### 9.4 矩阵聚合

按 family_id 与日期范围拉取记录后，按 `date(measured_at)` 与 `period` 分组，每组取**最新一条**作为该格子代表（防止同一时段同一天重测覆盖）。

### 9.5 报表 abnormal_days

某天有任意一条记录状态为 high/vhigh/low 即视为"异常天"。

## 12. 错误处理与日志

- 所有路由层异常通过 FastAPI exception handler 转为 `{ code, message }` JSON。
- 业务错误码全集见各 router 的"错误"列。
- 日志格式：`%(asctime)s %(levelname)s %(name)s [%(request_id)s] %(message)s`，写到 stdout（systemd 收集）+ `/var/log/glucose-api.log`。
- 每个请求注入 `request_id`（uuid4 短形），返回头 `X-Request-Id` 便于客服排查。
- LLM 调用失败、外呼失败必须记录 ERROR 级日志，含原始 text 与异常类。

无需缓存层、无需读写分离、无需消息队列。

## 14. 安全考虑

- JWT 密钥从环境变量加载，不入库不入代码。
- SQLite 文件权限 0600，仅服务用户可读写。
- 邀请码不在登录响应中泄漏（仅在 `/families/me` 给 creator 看到）。
- 微信 `code` 只在登录时一次性用，不持久化。
- LLM 调用前剥离敏感词检测：本场景不敏感，但代码留 hook。
- 日志不记录 raw token、不记录 openid 全量（截断后 4 位）。

## 15. 部署拓扑

### 15.1 Nginx 片段

```
location /api/ {
    proxy_pass http://127.0.0.1:8080/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_read_timeout 30s;
}
```

### 15.2 systemd 单元

```
[Unit]
Description=家有糖人 API
After=network.target

[Service]
User=glucose
WorkingDirectory=/opt/glucose-api
EnvironmentFile=/opt/glucose-api/.env
ExecStart=/opt/glucose-api/.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8080 --workers 1
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

### 15.3 备份

cron 每日 03:00 执行 `scripts/backup.sh`，将 SQLite 文件 sqlite3 `.backup` 拷到 `/var/backups/glucose/glucose-YYYYMMDD.db`，保留 30 天。
