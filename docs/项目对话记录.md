**沙箱实测路径**（一次 bash 调用内完成全套）：

3. `GET /api/v1/health` → 200，X-Request-Id `908aafe5`
4. `POST /auth/login` → 200，token 长度 179
5. `POST /families`（创建"咱家"）→ invite_code `BZYEMC`，role_of_me=creator

7. `POST /ai/parse-record`（"今天早上 9:20 空腹血糖 11.1，吃了二甲双胍"）→ 4 字段全识别，missing=[]
8. uvicorn 访问日志清晰，每条 SQL BEGIN/ROLLBACK 配对，所有 endpoint 200 OK

**结论**：后端代码完整可运行，等用户在 Mac 上 `bash scripts/dev_run.sh` 启动后接入小程序联调。

### 9. 调试时最可能踩的坑

1. **微信工具不校验合法域名没勾**：所有请求都会失败，控制台报 `url not in domain list`
2. **后端没启动 / 端口被占**：`lsof -i :8080` 看占用，杀掉或换端口（同步改 `const.js`）
3. **CORS 报错**：`.env` 的 `CORS_ORIGINS` 缺 `http://localhost`，加上重启
4. **改后端代码没生效**：`uvicorn --reload` 应自动重载，没生效就 Ctrl+C 重启
5. **JWT 过期 / SECRET 不一致**：dev_run.sh 生成的 `.env` 里 `JWT_SECRET` 是固定的，重启不影响 token，但**删掉 .env 重新跑会换 SECRET 导致旧 token 全部失效**
6. **多家人混用同一 mock openid**：默认 mock 是 `local_dev_mama`，所有登录都映射到同一 user，看起来"家庭只有妈妈一个人"，需要切 `WX_MOCK_OPENID` 模拟其他家人
